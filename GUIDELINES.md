# Code of Conduct

## Our Commitment

KeepMoviEZ is an open-source project, and we value contributions from developers of all skill levels and backgrounds. We are committed to providing a professional and productive environment for everyone who participates.

## Expected Behavior

As a contributor or participant in this project, you are expected to:

- **Be Professional**: Communicate respectfully in all project interactions
- **Be Constructive**: Provide helpful feedback and accept criticism gracefully
- **Be Collaborative**: Work together toward improving the project
- **Be Respectful**: Value different perspectives and experiences
- **Be Helpful**: Assist newcomers and answer questions when you can
- **Stay Focused**: Keep discussions relevant to the project and its goals

## Unacceptable Behavior

The following behaviors are not acceptable in this project:

- **Disrespectful Communication**: Insults, personal attacks, or inflammatory language
- **Harassment**: Any form of harassment, intimidation, or unwelcome behavior
- **Spam**: Off-topic spam, self-promotion unrelated to the project, or repetitive messages
- **Bad Faith Participation**: Trolling, deliberately derailing discussions, or intentionally wasting others' time
- **Privacy Violations**: Sharing others' private information without consent
- **Plagiarism**: Submitting others' work as your own

## Project-Specific Guidelines

Since KeepMoviEZ is a movie tracking application:

- **No Piracy**: Do not share, promote, or discuss illegal content sources
- **Stay On Topic**: Keep discussions focused on development, features, and improvements
- **Respect the Vision**: Ensure contributions align with KeepMoviEZ as a personal movie tracking tool
- **Be Patient**: This is maintained by volunteers; responses may take time

## Enforcement Responsibilities

Project maintainers are responsible for clarifying and enforcing our standards of acceptable behavior and will take appropriate and fair corrective action in response to any behavior that they deem inappropriate, threatening, offensive, or harmful.

Project maintainers have the right and responsibility to remove, edit, or reject comments, commits, code, wiki edits, issues, and other contributions that are not aligned with this Code of Conduct, and will communicate reasons for moderation decisions when appropriate.

## Scope

This Code of Conduct applies to:

- GitHub repository (issues, pull requests, discussions, code reviews)
- Project-related communication channels
- Official representation of the project in public spaces

## Reporting Issues

If you encounter behavior that violates this Code of Conduct:

**Preferred Method**: Open a GitHub Issue with the label `code-of-conduct`

For sensitive matters, you may contact the maintainer directly, but GitHub Issues are recommended for transparency.

**What to Include**:

- Clear description of the incident
- Links or references to the problematic behavior
- Any relevant context
- (Optional) Whether you prefer to remain anonymous

## Enforcement

Reports will be reviewed by the project maintainer. Actions may include:

1. **Warning**: Private notification about the violation
2. **Temporary Restriction**: Limited ability to participate (e.g., issue commenting)
3. **Permanent Ban**: Removal from the project

The project maintainer will determine the appropriate response based on the severity and context of the violation.
